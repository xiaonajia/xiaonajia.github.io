IE中背景透明元素无法触发hover
最近，遇到了一个IE浏览器下的兼容性的bug，产品需求如下图：

举个例子，代码如下：
<style type="text/css">
	.bubble_hover{
		cursor: pointer;
		color: blue;
		padding-left: 40px;
		width: 130px;
	}
	.bubble_con{
		display: none;
		padding: 10px;
		border: 1px solid blue;
		width: 100px;
		margin: 10px 0;
	}
	.bubble_hover:hover + .bubble_con{
		display: block;
	}
</style>
<div class="bubble">
	<div class="bubble_hover"></div>
	<div class="bubble_con">这里是内容</div>
</div>
效果如下：
鼠标悬停
这里是内容
问题描述：

在大部分的浏览器中，都可以正常显示，除了IE。

IE中，鼠标悬停，发现，啥都没有，根本不带动的啊，于是，F12，看看元素，发现元素也有啊，可是为啥不动嘞？查了查资料，发现：

解决办法：

网上很多说法说的是，背景透明，或者不加背景，无法触发hover伪类，貌似这是个很经典的IEbug呀，涨姿势了~

.bubble_hover{
	cursor: pointer;
	color: blue;
	padding-left: 40px;
	width: 130px;
	background: #f00; /*加背景*/
}
